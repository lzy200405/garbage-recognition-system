认证服务接口文档
基本信息
基础URL: /auth

全局返回格式: 所有接口均返回统一的Result格式

全局响应格式说明
json
{
"code": 200,
"msg": "success",
"data": null
}
1. 用户注册
   接口信息
   接口描述: 新用户注册

请求路径: /auth/register

请求方法: POST

Content-Type: application/json

请求参数
参数名	类型	必填	描述
username	string	是	用户名
password	string	是	密码
email	string	否	邮箱
phone	string	否	手机号
请求示例
json
{
"username": "zhangsan",
"password": "123456",
"email": "zhangsan@example.com",
"phone": "13800138000"
}
响应示例
注册成功
json
{
"code": 200,
"msg": "success",
"data": "注册成功"
}
注册失败（参数缺失）
json
{
"code": 500,
"msg": "用户名或密码不能为空",
"data": null
}
注册失败（用户名已存在）
json
{
"code": 500,
"msg": "用户名已存在",
"data": null
}
2. 用户登录
   接口信息
   接口描述: 用户登录，获取认证Token

请求路径: /auth/login

请求方法: POST

Content-Type: application/x-www-form-urlencoded

请求参数
参数名	类型	必填	描述
username	string	是	用户名
password	string	是	密码
请求示例
text
POST /auth/login
Content-Type: application/x-www-form-urlencoded

username=zhangsan&password=123456
响应示例
登录成功
json
{
"code": 200,
"msg": "success",
"data": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9..."
}
登录失败
json
{
"code": 500,
"msg": "用户名或密码错误",
"data": null
}
使用说明
登录成功后返回的Token需要在后续请求中通过Header传递：

Header名称: satoken 或 Authorization

Header值: 返回的token字符串

3. 用户登出
   接口信息
   接口描述: 用户退出登录

请求路径: /auth/logout

请求方法: GET

权限要求: 需要登录（需携带Token）

请求头
参数名	类型	必填	描述
satoken	string	是	登录返回的Token
响应示例
登出成功
json
{
"code": 200,
"msg": "success",
"data": "退出成功"
}
登出失败（未登录）
json
{
"code": 401,
"msg": "无效Token或未登录",
"data": null
}
状态码说明
状态码	说明
200	请求成功
401	未认证或Token无效
500	业务处理失败
